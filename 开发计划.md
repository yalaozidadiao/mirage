# 米拉奇战记开发计划

## 项目定位

《米拉奇战记》当前是一个纯前端 Canvas 即时策略游戏。v4.0 已在 v3.4 refactor 基线上加入本地双人、好友房间联机和房主权威状态同步。联机技术方向为 P2P + PeerJS + 房主权威。

项目优先级：先保证可玩和稳定，再追求联机体验、部署便利和公平性。

## v4.0 发布说明

- 面向玩家的模式入口统一为“联机对战”，不再使用阶段性测试措辞。
- 正式发布文件为 `米拉奇战记v4.0.html`，`index.html` 自动跳转到该版本。
- 房主创建房间并发送房主 ID，客机输入 ID 加入后由房主开始对战。
- 联机采用房主权威状态同步；校园网、公司网等严格网络可能需要更换手机热点或家庭网络。
- GitHub Pages 发布时推荐提供根目录入口或明确的 HTML 文件链接。

## 开发原则

1. `main` 始终保持稳定可玩。
2. 大功能在独立分支开发，例如 `online-mvp`。
3. 每个阶段都要有明确可验收结果。
4. 第一阶段不急于接网络，先把输入、状态、模式边界理清。
5. 联机逻辑以后以“房主为权威”，客户端只提交操作并渲染结果。
6. 后续开发沟通、计划说明、验收说明、UI 卡片文案优先使用中文。
7. 第三方前端依赖优先随项目放入 `vendor/`，避免玩家运行时依赖外部 CDN。

## 阶段一：本地双人与联机前置重构（已完成）

目标：在不引入网络的情况下，让游戏支持蓝红双方都由本地玩家控制。

主要任务：

- 创建 `online-mvp` 分支。
- 增加统一命令入口 `submitCommand(command)`。
- 将 UI 点击出兵改为提交命令。
- 增加 `single` 和 `localVersus` 两种对局类型。
- 单人模式保持蓝方玩家、红方 AI。
- 本地双人模式关闭 AI，红方卡片可操作。
- 调整胜负文案，区分 AI 对战和玩家对战。
- 保持旧版本文件不删除。

验收标准：

- 原有单人模式全部可玩。
- 本地双人模式下双方都能出兵。
- 红方 AI 在本地双人中不会自动出兵。
- 双方经济、基地血量、胜负结算正常。
- 控制台无明显错误。

## 阶段二：PeerJS 连接 MVP（已完成）

目标：实现好友之间通过房间 ID 建立 P2P 连接。

主要任务：

- 引入 PeerJS 客户端。
- 增加联机对战模式入口。
- 支持创建房间、复制房主 ID、加入房间、断开连接。
- 建立基础消息通道：`hello`、`welcome`、`startGame`、`command`、`commandAccepted`、`stateDigest`、`error`。
- 房主固定为蓝方，客机固定为红方。
- 房主开始联机对战后，双方进入 `onlineVersus` 模式。
- 客机出兵命令发送给房主，由房主校验并确认。
- 房主低频发送简要状态摘要，暂不做完整单位快照同步。
- PeerJS 客户端文件放在 `vendor/peerjs.min.js`，HTML 使用本地脚本引用。

验收标准：

- 两个浏览器窗口可以连接。
- 客机可以向房主发送出兵命令。
- 房主能接收并处理命令。
- 基础断线提示可见。
- 单人模式和本地双人模式不受影响。

正确测试方式：

1. 推荐使用 GitHub Pages 测试。玩家只需要打开网页，不需要单独下载 PeerJS。
2. 本地测试时推荐启动 HTTP 服务，不推荐直接双击 HTML 用 `file://` 打开。
3. 在项目根目录执行：`python -m http.server 8000`。
4. 浏览器访问：`http://localhost:8000/米拉奇战记v3.4-refactor.html`。
5. 开两个浏览器窗口或两台设备访问同一个页面。
6. 房主选择“联机对战”并点击“创建房间”，复制房主 ID。
7. 客机选择“联机对战”，输入房主 ID，点击“加入房间”。
8. 房主看到客机连接后点击“开始联机对战”。
9. 验收重点：客机红方出兵后，房主画面应出现红方单位；状态摘要应持续更新。

## 阶段三：房主权威同步（已完成）

目标：房主运行完整战斗逻辑，客机只提交操作并根据房主状态渲染。

主要任务：

- 房主校验所有出兵命令。
- 房主以约 10 次/秒广播完整 `stateSnapshot`。
- `stateSnapshot` 包含双方玩家状态、单位数组、经济计时器、收入、胜负、轮次、暂停和速度状态。
- 客机不运行完整战斗模拟。
- 客机根据快照更新画面和 UI。
- 客机发送出兵命令后等待房主快照，不再本地买兵。
- 联机客机禁用暂停和 2 倍速，暂停和速度由房主控制。
- 保留 `stateDigest` 作为联机状态栏摘要。

验收标准：

- 双方看到的基地血量、金币、单位大体一致。
- 延迟存在时仍能正常对战。
- 非房主不能通过本地修改直接影响权威状态。
- 客机刷新/断开时不影响房主本地战斗继续。
- 单人模式和本地双人模式不受影响。

## 阶段四：部署与稳定性（后续）

目标：让外部玩家可以较稳定地访问和测试。

主要任务：

- 使用 GitHub Pages 部署前端。
- 评估是否自建 PeerServer。
- 评估 TURN 服务需求。
- 增加版本号和协议版本校验。
- 增加错误提示和连接状态显示。

验收标准：

- 通过公开链接可以打开游戏。
- 好友可通过房间 ID 进行测试。
- 常见连接失败有明确提示。

## 暂不处理

- 排位、账号、匹配系统。
- 严格反作弊。
- 观战系统。
- 录像回放。
- 大规模多人房间。
